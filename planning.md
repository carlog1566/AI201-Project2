### Tool 1: search_listings

**What it does:**
This tool is used to search for the top matching items based on the users query and the listings available. It will return the top result, or tells the user to try a different query if there are no matching listings.

**Input parameters:**
<!-- List each parameter, its type, and what it represents -->
- `description` (str): The name of the desired item
- `size` (str): The size the user wants
- `max_price` (float): The max amount of money the user is willing to spend

**What it returns:**
The top matching listing as a dictionary containing fields like id, title, price, and other listing details.

**What happens if it fails or returns nothing:**
If no listings match, the agent tells the user to adjust their search criteria since no matching results were found and the remaining tools are not called.

---

### Tool 2: suggest_outfit

**What it does:**
This tool is used to recommend how to style the selected item using clothes already in the user's wardrobe.

**Input parameters:**
<!-- List each parameter, its type, and what it represents -->
- `new_item` (dict): The listing selected from search_listings
- `wardrobe` (dict): The user's waardrobe containing an items list

**What it returns:**
Recommendation text describing which wardrobe items would pair well with the new item and how to style them.

**What happens if it fails or returns nothing:**
The agent will provide general styling advice with the new item alone.

---

### Tool 3: create_fit_card

**What it does:**
This tool generates a short, social media style caption based ont he recommended outfit and the newly selected item.

**Input parameters:**
<!-- List each parameter, its type, and what it represents -->
- `outfit` (...): The outfit recommendation generated by suggest_outfit.

**What it returns:**
A social media style caption that highlights the thrift find and completed look.

**What happens if it fails or returns nothing:**
The agent will genereate a caption using only the new item or asks for more information.

## Planning Loop

**How does your agent decide which tool to call next?**
<!-- Describe the logic your planning loop uses. What does it look at? What conditions change its behavior? How does it know when it's done? -->
1. The agent always begins by calling search_listings() using the user's query, size, preference, and budget.
2. If no listings are returned, it immediately stops and returns a mesage suggesting the user broaden their search.
3. If listings are found, the agent stores the top result as selected_item.
4. The agent will then call suggest_outfit() using that item and the user's wardrobe.j
5. The returned recommendation is then saved.
6. The agent finally calls create_fit_card() using the outfit suggestion and selected item.
7. The caption is then saved.
8. The process then ends after all three outputs have been generated and returned back to the user.

## State Management

**How does information from one tool get passed to the next?**
<!-- Describe how your agent stores and accesses state within a session. What data is tracked? How is it passed between tool calls? -->
The agent stores information in a session dictionary as it moves through the workflow. It tracks the selected item from search_listings, the outfit suggestion from suggest_outfit, and the final caption from create_fit_card. Each tool uses information produced by the previous step, allowing the agent to build the final response incrementally.

## A Complete Interaction (Step by Step)

The user asks, "I'm looking for a vintage graphic tee under $30. I mostly wear baggy jeans and chunky sneakers. What's out there and how would I style it?", where FitFindr first calls search_listings, which finds matching items and selects the top result; if there are no matching listings, FitFindr stops the process and suggests the user adjust their criteria. It then calls suggest_outfit using the selected tee and the user's wardrobe, giving recommendations on what clothes from their wardrobe would pair well with the selected tee. Finally, create_fit_card would generate a caption that is inspired by the outfit.

**Example user query:** "I'm looking for a vintage graphic tee under $30. I mostly wear baggy jeans and chunky sneakers. What's out there and how would I style it?"

**Step 1:**
The agent calls search_listings("vintage graphic tee", max_price=30.0) to find matching items. It then returns the top listing.

**Step 2:**
The agent passes the selected listing and the user's wardrobe to suggest_outfit(), which will return a recommendation on how to style the selected item and which clothes from the user's wardrobe would pair well with it.

**Step 3:**
The agent calls create_fit_card() using the outfit suggestion and the selected item to genereate a short social media style caption.

**Final output to user:**
The user sees the recommended listing, an outfit suggestion using pieces from their wardrobe, and a caption they can use to share the look.
